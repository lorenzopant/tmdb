## 🚀 1.17.0 - 27 March 2026

#### ✨ New

- [People API](/docs/api-reference/people) — full first-class support for TMDB person-specific endpoints via `tmdb.people` (details, credits, images, translations, external IDs, and more)
- [Authentication guide](/docs/getting-started/authentication) — new documentation covering Bearer token vs v3 API key usage, detection logic, and environment variable setup

#### 🔐 Auth

- The client now supports both **Bearer tokens** (JWT Read Access Tokens) and **v3 API keys** — the correct auth method is detected automatically based on the token structure
- JWT tokens continue to be sent as `Authorization: Bearer <token>`
- Plain v3 API keys are now correctly appended as `?api_key=<key>` query parameter instead of being sent (incorrectly) as a Bearer header

#### 🐞 Bug Fixes

- Fixed broken links in [TV Series Lists](/docs/api-reference/tv-series-lists) API docs and related type pages (404s caused by incorrect route paths)
- Fixed `isJwt()` incorrectly returning `false` for expired tokens, which would have caused a valid JWT to be silently sent as an `api_key` query parameter (leaking it in URLs) instead of as a Bearer header
- Unit tests no longer throw at startup when `TMDB_BEARER_TOKEN` is not set in the environment — they use a hardcoded mock JWT since `fetch` is fully mocked

## 🚀 1.16.0 - 23 March 2026

#### ✨ New

- [People Lists API](/docs/api-reference/people-lists) — get a paginated list of people ordered by TMDB popularity
- [Reviews API](/docs/api-reference/reviews) — fetch full details of a single TMDB review by its review ID

#### 🧪 Tests

- Split tests into unit and integration tests for easier contributions
- Added more unit tests

#### 🐞 Bug Fixes

- `include_image_language` now accepts an array of languages (e.g. `["en", "null"]`) instead of a comma-separated string
- `series_id` was incorrectly destructured and being passed into query params
- `withDefaults()` returning `undefined` when passed `undefined` as value
