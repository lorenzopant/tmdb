They are shared between movie and TV series responses.

### `KnownForItem`

A union type representing either a movie or TV show in a person's `known_for` list.
Use the `media_type` discriminator to narrow the type.

```ts
type KnownForItem = KnownForMovie | KnownForTV;
```

### `MediaType`

A discriminator union for media entries that can be either a movie or a TV show.

```ts
type MediaType = "movie" | "tv";
```

## Params

`TMDBQueryParams` using `Pick`, ensuring consistency and reducing duplication.

### `WithParams`

A generic utility type that picks one or more keys from [`TMDBQueryParams`](#tmdbqueryparams).
Used internally to construct all the `With*` types above.

```ts
type WithParams<T extends keyof TMDBQueryParams> = Pick<TMDBQueryParams, T>;
```

## Pagination

The `PaginatedResponse` type wraps all list-based API responses that support pagination.

### `PaginatedResponse<T>`

A generic wrapper returned by any endpoint that paginates its results.
The `results` array type varies per endpoint â€” refer to each method's return type for the specific item type.

<TypeTable
	type={{
		page: {
			description: "The current page number.",
			type: "number",
		},
		total_pages: {
			description: "Total number of pages available.",
			type: "number",
		},
		total_results: {
			description: "Total number of results across all pages.",
			type: "number",
		},
		results: {
			description: "Array of items for the current page.",
			type: "T[]",
		},
	}}
/>

<Callout type="info">
	TMDB caps pagination at **500 pages** regardless of `total_pages`. Requests beyond page 500 will return an error.
</Callout>
