---
title: Interceptors
description: Hook into every request before it fires and transform every response globally before it reaches the caller.
---

The TMDB client supports two interceptor hooks:

- **Request interceptors** — run before every HTTP request (modify params, headers, or endpoint)
- **Response interceptors** — run after every successful response, before data reaches the caller (transform or inspect data)

---

## Request interceptors

Each request interceptor receives a [`RequestContext`](#requestcontext) and must return the (possibly modified) context.

### Registering

```ts
import { TMDB } from "@lorenzopant/tmdb";

// At construction time
const tmdb = new TMDB("your-api-key", {
	interceptors: {
		request: [(ctx) => ({ ...ctx, params: { ...ctx.params, include_adult: false } })],
	},
});

// Or at runtime — returns `this` so calls can be chained
tmdb.use("request", (ctx) => ({ ...ctx, params: { ...ctx.params, include_adult: false } })).use("request", async (ctx) => {
	const token = await refreshToken();
	return { ...ctx, headers: { ...ctx.headers, Authorization: `Bearer ${token}` } };
});
```

### Examples

**Force-disable adult content**

```ts
tmdb.use("request", (ctx) => ({
	...ctx,
	params: { ...ctx.params, include_adult: false },
}));
```

**Dynamically refresh the access token (server-side)**

```ts
tmdb.use("request", async (ctx) => {
	const token = await getValidToken();
	return { ...ctx, headers: { ...ctx.headers, Authorization: `Bearer ${token}` } };
});
```

**Inject a custom header (server-side only)**

```ts
tmdb.use("request", (ctx) => ({
	...ctx,
	headers: { ...ctx.headers, "X-App-Version": "2.0.0" },
}));
```

<Callout type="warn">
  **Browser / CORS warning** — Adding non-standard headers (e.g. `X-*`) triggers a CORS preflight
  (`OPTIONS`) request. The TMDB API only allows `Authorization` and `Content-Type` in
  `Access-Control-Allow-Headers`, so custom headers will be **blocked by the browser**.
  Header injection is safe **only in server-side environments** (Node.js, edge runtimes, etc.).

Modifying `params` or `endpoint` is safe in all environments.

</Callout>

## Response interceptors

Each response interceptor receives a [`ResponseContext`](#responsecontext) — containing the sanitized data, the originating request context, and the HTTP status code — and must return the (possibly transformed) data.

Response interceptors run **only for successful responses**. Errors bypass them entirely.

### Registering

```ts
// At construction time
const tmdb = new TMDB("your-api-key", {
	interceptors: {
		response: [
			(ctx) => {
				console.log(`[${ctx.status}] ${ctx.request.endpoint}`);
				return ctx.data;
			},
		],
	},
});

// Or at runtime — returns `this` so calls can be chained
tmdb.use("response", (ctx) => {
	console.log(ctx.data);
	return ctx.data;
}).use("response", async (ctx) => {
	/* async supported */ return ctx.data;
});
```

### Examples

**Log every response**

```ts
tmdb.use("response", (ctx) => {
	console.log(`[${ctx.status}] ${ctx.request.endpoint}`, ctx.data);
	return ctx.data;
});
```

**Add a computed field to every response**

```ts
tmdb.use("response", (ctx) => ({
	...(ctx.data as object),
	_fetched_at: new Date().toISOString(),
}));
```

**Send every response to an analytics service**

```ts
tmdb.use("response", async (ctx) => {
	await analytics.track({ endpoint: ctx.request.endpoint, status: ctx.status });
	return ctx.data;
});
```

## Execution order

```
Request interceptors (registered order)
        ↓
   HTTP fetch
        ↓
Response interceptors (registered order)
        ↓
     Caller
```

Construction-time interceptors always run before runtime `use()` interceptors.
If any interceptor throws, the error propagates and subsequent interceptors are skipped.

```ts
const tmdb = new TMDB(token, {
	interceptors: {
		request: [reqA], // runs 1st
		response: [resA], // runs 1st for responses
	},
});

tmdb.use("request", reqB); // runs 2nd
tmdb.use("response", resB); // runs 2nd for responses
```

### `RequestContext`

		},
	}}
/>

```ts
type RequestInterceptor = (context: RequestContext) => RequestContext | Promise<RequestContext>;
```

### `ResponseContext<T>`

<TypeTable
	type={{
		data: {

```ts
type ResponseInterceptor = <T>(context: ResponseContext<T>) => T | Promise<T>;
```
